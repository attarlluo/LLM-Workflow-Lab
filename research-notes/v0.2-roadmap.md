# v0.2 Roadmap — Cognitive Protocols & Academic Mapping

**版本目标：**  
从“基础结构”进入“方法论初步成型”阶段。  
v0.1 打了地基，v0.2 的任务是让研究正式开始增长血肉。

---

## 1. Version Goals（版本目标）

### **1.1 建立首批认知协议（protocol draft）**
目标不是完美，而是：
- 抓住几个结构模板
- 明确 pipeline 的序列
- 建第一批“可复用但不稳定”的 protocol

建议先做三类：

1. **Task Decomposition Protocol v0.1**  
   - 输入澄清层  
   - 子任务分类层  
   - 推理路径选择层  

2. **Uncertainty Protocol v0.1**  
   - 置信度标注规则  
   - 自检触发条件  
   - critique → verify → rewrite 流程  

3. **Multi-pass Reasoning Protocol v0.1**  
   - 什么时候需要第二次推理  
   - 什么时候需要 parallel thought  
   - 如何 merge multiple passes  

这些将是后期方法论体系的骨架。

---

## 2. Academic Mapping（学术映射任务）

v0.2 要建立项目的“理论基础地板”。

### **2.1 初始论文映射（至少三篇）：**
- **Chain-of-Thought (Wei et al., 2022)**  
- **ReAct: Reason + Act (Yao et al.)**  
- **Reflexion (Shinn et al.)**

每篇 mapping 包含：
- 要点  
- 流程  
- 本研究可借鉴点  
- 是否能转为 protocol  
- 与目前结构的关系  

目标：  
将直觉 → 理论 → protocol 的路径跑通一次。

---

## 3. Patterns Prototyping（初步模式建模）

建立两到三个“可复用但不稳定”的 workflow pattern：

建议从以下选两个：

1. **Ambiguous-task clarification pattern**  
2. **Structured output enforcement pattern**  
3. **Error-detection pattern（LLM 自检）**  
4. **Multi-pass merge pattern**

这里不求完美，求雏形。

---

## 4. Fragments Consolidation（碎片桶整理流程）

建立一个 weekly 固定动作：

1. 清点 fragments 的新增内容  
2. 分类：notes / protocol / mapping / discard  
3. 抽取 1–2 个 insight 写短文档  
4. 更新 v0.2-roadmap 的进度标记  

这一步确保研究链条不会中断。

---

## 5. Deliverables（v0.2 的产出）

### 必须包含：
- 3 个 protocol 草稿  
- 3 个 paper mapping  
- 2–3 个 workflow pattern 雏形  
- 每周碎片整合至少一次  
- 1 个月末总结（用于 v0.3）

### 可选增强：
- 第一个 “methodology skeleton” 文件草稿  
- 早期可视化流程图  

---

## 6. Version Flow（版本运行方式）

v0.2 会运行大约 2–4 周。  
流程如下：

1. 持续积累碎片  
2. 每周消化  
3. 逐步写出草稿  
4. 推进 mapping  
5. 修正协议  
6. 在 formal-docs 建立最早的 stable drafts

最终目标：  
**让本项目正式具有“方法论创作能力”。**

---

## 7. Risks / 注意事项

- 不要追求完美（v0.3 才是优雅化版本）  
- 不要把所有事一次写完（你会写到怀疑人生）  
- mapping 一定要做，不然 protocol 会飘  
- 不要推太复杂的 protocol，一次只搞一条链  
- 保持每周至少 2 次小 commit
